# 테이블 매핑 수정 가이드

## 문제
Prisma가 `public.Novel` (대문자) 테이블을 찾으려 하지만, 실제 DB에는 `novel` (소문자) 테이블이 존재함.

## 원인
Prisma 스키마에서 모델 이름(`Novel`)과 실제 테이블 이름(`novel`)이 다를 때, `@@map` 어노테이션으로 매핑해야 합니다.

## 적용된 수정 사항

다음 모델들에 `@@map` 추가:

1. **Novel** → `@@map("novel")`
2. **NovelComment** → `@@map("novelComment")`
3. **NovelCommentLike** → `@@map("novelCommentLike")`
4. **NovelEpisode** → `@@map("novelEpisode")`
5. **NovelGenre** → `@@map("novelGenre")`
6. **NovelLike** → `@@map("novelLike")`

## 다음 단계

Prisma 클라이언트를 재생성해야 합니다:

```bash
npx prisma generate
```

또는 개발 서버를 재시작하면 자동으로 재생성됩니다.

## 확인 방법

재생성 후, 브라우저에서 소설 목록 페이지를 새로고침하면 정상적으로 표시되어야 합니다.

## 참고

- `User` 모델은 이미 `@@map("user")`가 있어서 정상 작동했습니다.
- 다른 모델들도 같은 문제가 있을 수 있으니, 비슷한 오류가 발생하면 해당 모델에 `@@map` 추가가 필요합니다.
